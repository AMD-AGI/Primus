Code Review // Critical findings

Cross-checked against DeepSeek-V4-Flash/inference/model.py, the transformers.models.deepseek_v4 reference and NeMo Automodel's V4 implementation. Severity tags are CRIT (architecture parity), HIGH (distributed / spec correctness), MED (reuse / hygiene). Full notes in plan-2/00-review-findings.md.

Separate K and V projections

Real V4 has a single wkv latent (K = V = kv). Wastes parameters and blocks checkpoint loading.

Missing q_norm and kv_norm

HF / inference reference applies a per-head q_rms after wq_b and a kv_norm after wkv. Both are absent in Primus.

No grouped low-rank O projection

Real V4 has wo_a + wo_b with o_groups=8, o_lora_rank=1024. Primus uses a flat linear_o_proj.

HashRouter has no learnable gate

Real V4 hash gate gathers learned scores at tid2eid indices. Primus emits uniform 1/topk weights.

Wrong clamped SwiGLU

HF reference clamps gate / up before the multiply. Primus clamps the post-multiply output AND fuses w1/w3.

No state-dict adapter

Primus model cannot consume DeepSeek-V4-Flash safetensors for finetune / numerical alignment.

Custom block / attention bypass MLA + TransformerBlock

V4 attention is a plain nn.Module; block does not subclass TransformerBlock. Loses MLA backends, recompute, CUDA graph, sequence-parallel.

HC × PP boundary breaks the math

HyperHead is applied at the end of every PP stage and re-expanded on the next; K-stream context is destroyed.

Anti-patterns in distributed forward

Token-IDs stashed on decoder._v4_token_ids, position_ids replaced with arange(S), projections built with parallel_mode="duplicated" (no TP).

MTP not on Megatron's MTP path

Custom DeepseekV4MTPBlock is opt-in; default training has no MTP loss.

Spec / provider hygiene

Provider re-instantiated, activation_func returns a class, compress_ratios is a YAML string, scaling_factor ignored, etc.

Reuse / hygiene gaps

Local _RMSNorm duplicates, custom YaRN re-implementation, custom dual-RoPE, mixed concerns inside one 778-line block file.
