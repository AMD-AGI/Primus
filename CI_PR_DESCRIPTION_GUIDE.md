# 在 CI 中处理 PR Description 的方案

## 方案概述

根据你的需求，有以下几种方式可以在 CI 中将 PR description 写入 commit：

---

## 🎯 方案 1：在 PR 合并时自动包含 Description（推荐）

### GitHub 仓库设置

在 GitHub 仓库设置中启用：

1. **Settings** → **General** → **Pull Requests**
2. 勾选 **"Allow squash merging"**
3. 选择 **"Default to pull request title and description"**

这样在 squash merge 时会自动包含 PR description。

---

## 🎯 方案 2：在 CI 中自动添加 PR Description 到新 Commit

### 添加到 `ci.yaml` 的 `run-unittest` job：

```yaml
- name: Auto Commit PR Description
  if: github.event_name == 'pull_request'
  run: |
    # 获取 PR 信息
    PR_NUMBER="${{ github.event.pull_request.number }}"
    PR_TITLE="${{ github.event.pull_request.title }}"
    PR_BODY="${{ github.event.pull_request.body }}"
    PR_AUTHOR="${{ github.event.pull_request.user.login }}"

    # 创建包含 PR description 的文件
    cat > PR_INFO.md << EOF
    # Pull Request #${PR_NUMBER}

    **Title:** ${PR_TITLE}
    **Author:** ${PR_AUTHOR}
    **URL:** ${{ github.event.pull_request.html_url }}

    ## Description

    ${PR_BODY}

    ---
    Generated by CI on $(date)
    EOF

    # 配置 git
    git config user.name "GitHub Actions Bot"
    git config user.email "actions@github.com"

    # 添加并提交
    git add PR_INFO.md
    git commit -m "docs: add PR #${PR_NUMBER} description

    ${PR_BODY}

    Co-authored-by: ${PR_AUTHOR} <${PR_AUTHOR}@users.noreply.github.com>"

    # 推送到当前分支（需要权限）
    git push origin HEAD:${{ github.head_ref }}
```

## 🎯 方案 3：保存 PR Description 到 Artifact（不修改 commit）

### 添加到 CI workflow：

```yaml
- name: Save PR Description
  if: github.event_name == 'pull_request'
  run: |
    PR_NUMBER="${{ github.event.pull_request.number }}"
    PR_TITLE="${{ github.event.pull_request.title }}"
    PR_BODY="${{ github.event.pull_request.body }}"

    mkdir -p pr_metadata

    # 保存为 JSON
    cat > pr_metadata/pr_${PR_NUMBER}.json << 'EOF'
    {
      "number": ${{ github.event.pull_request.number }},
      "title": "${{ github.event.pull_request.title }}",
      "author": "${{ github.event.pull_request.user.login }}",
      "url": "${{ github.event.pull_request.html_url }}",
      "body": $(echo '${{ github.event.pull_request.body }}' | jq -Rs .),
      "created_at": "${{ github.event.pull_request.created_at }}",
      "updated_at": "${{ github.event.pull_request.updated_at }}"
    }
    EOF

    # 保存为 Markdown
    cat > pr_metadata/pr_${PR_NUMBER}.md << EOF
    # PR #${PR_NUMBER}: ${PR_TITLE}

    - **Author**: ${{ github.event.pull_request.user.login }}
    - **URL**: ${{ github.event.pull_request.html_url }}
    - **Created**: ${{ github.event.pull_request.created_at }}

    ## Description

    ${PR_BODY}
    EOF

- name: Upload PR Metadata
  if: github.event_name == 'pull_request'
  uses: actions/upload-artifact@v4
  with:
    name: pr-metadata-${{ github.event.pull_request.number }}
    path: pr_metadata/
    retention-days: 90
```

## 🎯 方案 4：在 Merge 时通过 GitHub Action 自动处理

### 创建新的 workflow: `.github/workflows/auto-merge-with-description.yaml`

```yaml
name: Auto Merge with PR Description

on:
  pull_request:
    types: [closed]

jobs:
  update-merge-commit:
    if: github.event.pull_request.merged == true
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Update Merge Commit Message
        run: |
          # 获取最新的 merge commit
          MERGE_COMMIT=$(git rev-parse HEAD)

          # 获取 PR 信息
          PR_NUMBER="${{ github.event.pull_request.number }}"
          PR_TITLE="${{ github.event.pull_request.title }}"
          PR_BODY="${{ github.event.pull_request.body }}"

          # 创建新的 commit message
          NEW_MESSAGE="${PR_TITLE}

${PR_BODY}

PR #${PR_NUMBER}"

          # 修改最后一个 commit 的 message（需要 force push 权限）
          git commit --amend -m "${NEW_MESSAGE}"

          # 注意：这需要 force push，谨慎使用
          # git push --force-with-lease origin main

          echo "Updated commit message with PR description"
```

## 🎯 方案 5：在测试通过后添加 PR Description Tag

### 添加到现有的 `run-unittest` job：

```yaml
- name: Tag with PR Description
  if: github.event_name == 'pull_request' && success()
  run: |
    PR_NUMBER="${{ github.event.pull_request.number }}"
    PR_TITLE="${{ github.event.pull_request.title }}"

    # 创建 git note 记录 PR description
    git config user.name "GitHub Actions Bot"
    git config user.email "actions@github.com"

    # 将 PR description 添加到 git notes
    git notes add -m "PR #${PR_NUMBER}: ${PR_TITLE}

${{ github.event.pull_request.body }}

Tested and approved by CI on $(date)"

    # 推送 notes（需要权限）
    git push origin refs/notes/*
```

## 📋 完整实现示例（推荐方案）

### 在 `ci.yaml` 的 `run-unittest` job 中添加：

```yaml
run-unittest:
  env:
    PRIMUS_WORKDIR: /apps/tas/0_public/primus_k8s_ci
  needs: [code-lint]
  runs-on: [tas-k8s]
  steps:
    # ... existing steps ...

    - name: Run Shell Tests
      run: |
        echo "Running Primus CLI shell tests..."
        bash ./tests/runner/run_all_tests.sh

    # 新增：保存 PR 信息
    - name: Save PR Information
      if: github.event_name == 'pull_request'
      run: |
        PR_NUMBER="${{ github.event.pull_request.number }}"
        PR_TITLE="${{ github.event.pull_request.title }}"
        PR_BODY="${{ github.event.pull_request.body }}"
        PR_AUTHOR="${{ github.event.pull_request.user.login }}"
        PR_URL="${{ github.event.pull_request.html_url }}"

        # 创建 PR 信息目录
        mkdir -p pr_logs

        # 保存 PR 描述到文件
        cat > pr_logs/PR_${PR_NUMBER}_description.md << 'EOFMARKER'
        # PR #${PR_NUMBER}: ${PR_TITLE}

        **Author:** ${PR_AUTHOR}
        **URL:** ${PR_URL}
        **Status:** Tests Passed ✅
        **Date:** $(date)

        ## Description

        ${PR_BODY}

        ## CI Results

        - ✅ Code Lint: Passed
        - ✅ Unit Tests: Passed
        - ✅ Shell Tests: Passed
        EOFMARKER

        echo "PR description saved to pr_logs/PR_${PR_NUMBER}_description.md"

        # 可选：推送到特定分支保存历史

        # git config user.email "actions@github.com"
        # git checkout -B pr-logs
        # git add pr_logs/

    - name: Clean

        rm -rf ${PRIMUS_WORKDIR}/Primus-Turbo
```

## 🔧 权限配置

如果需要在 CI 中 commit 和 push，需要在 workflow 中配置权限：

### 在 workflow 文件顶部添加：

```yaml
name: Primus-CI-TAS

on:
  workflow_dispatch:
  push:
    branches:
      - main
  pull_request:

permissions:
  contents: write        # 允许写入内容
  pull-requests: write   # 允许写入 PR
  issues: write          # 允许写入 issues

env:
  PRIMUS_TURBO_COMMIT: ...
```

## ⚙️ GitHub API 方式（高级）

### 使用 GitHub API 更新 PR 或创建 commit：

```yaml
- name: Update PR with Description Archive
  if: github.event_name == 'pull_request'
  env:
    GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: |
    PR_NUMBER="${{ github.event.pull_request.number }}"

    # 使用 GitHub CLI 添加评论
    gh pr comment ${PR_NUMBER} --body "✅ Tests passed! PR description has been archived.

**Original Description:**
${{ github.event.pull_request.body }}

---
_This is an automated message from CI_"
```

## 📊 推荐实施方案

根据不同需求选择：

| 需求 | 推荐方案 | 复杂度 |
|------|---------|--------|
| 仅记录 PR 信息 | 方案 3 (Artifact) | ⭐ 简单 |
| Merge 时包含 Description | 方案 1 (GitHub 设置) | ⭐ 最简单 |
| 在分支上创建 commit | 方案 2 (Auto Commit) | ⭐⭐ 中等 |
| 历史追踪 | 方案 5 (Git Notes) | ⭐⭐⭐ 复杂 |
| 自动化 Merge | 方案 4 (Workflow) | ⭐⭐⭐ 最复杂 |

## ✅ 快速开始

**最简单的方式（无需修改 CI）：**

1. 去 GitHub 仓库设置
2. **Settings** → **Pull Requests**
3. 勾选 **"Default to pull request title and description"**
4. 使用 **Squash and merge** 合并 PR

这样每次合并 PR 时，commit message 会自动包含 PR description！
